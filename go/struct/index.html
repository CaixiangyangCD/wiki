    结构体¶

    Golang中没有"类"的概念，也不支持"类"的继承等面向对象的概念，Golang中通过结构体的内嵌，再配合接口

    比面向对象具有更高的扩展性和灵活性

    类型别名和自定义类型¶

    自定义类型¶

    在Golang中，有一些基础的数据类型，例如string、整型、浮点型、布尔类型等等，Golang可以使用type关键字来自定义类型

    自定义类型是定义了一个全新的类型，我们可以基于内置的基本类型定义，也可以通过struct定义：

    // 将MyInt定义为int类型
type MyInt int

    通过type关键字的定义，MyInt就是一种新的类型，它具有int的特性

    类型别名¶

    类型别名规定：TypeAlias只是Type的别名，本质上TypeAlias和Type是同一个类型，就像一个孩子小时候有小名、乳名，上学后用学名，英语老师又会给他起英文名，但是这些名字都是指的他本人

    type TypeAlias = Type

    我们之前见过的rune和byte就是类型别名，它们的定义如下：

    type byte = uint8
type rune = int32

    类型定义和类型别名的区别¶

    类型别名和类型定义表面上看只有一个等号的差异，通过下面这段代码就可以理解它们之间的却别

    package main

import "fmt"

// 类型定义
type NewInt int

// 类型别名
type MyInt = int

type MyNewInt = NewInt

func main() {
    // 定义a变量为NewInt类型
    var a NewInt

    // 定义b变量为MyInt类型
    var b MyInt

    // 定义c变量为MyNewInt类型
    var c MyNewInt

    // 打印a和b的类型
    fmt.Printf("a: %T\nb: %T\nc: %T", a, b, c)

}
    执行结果
    a: main.NewInt
b: int
c: main.NewInt

    结果显示a的类型是main.NewInt，表示main包下定义的NewInt类型，b的类型是int。

    MyInt类型只会在代码中存在，编译完成时并不会有MyInt类型

    结构体¶

    Golang中的基础数据类型可以表示一些事物的基本属性，但是当我们想要表达一个复杂的属性时，使用单一的基本数据类型已经无法满足需求了，Golang提供了一种自定义数据类型，可以封装多个基本数据类型，这种数据类型就叫做 结构体。

    定义¶

    使用type和struct关键字来定义结构体，语法格式如下：

    type 类型名称 struct {
    字段名 字段类型
    字段名 字段类型
    ...
}

    其中：

    • 类型名：标识自定义结构体的名称，在同一个包内不能重复
    • 字段名：表示结构体字段名，在结构体中字段名不能重复，字段名大写表示可公开访问，小写表示私有(仅在当前结构体的包中可使用)
    • 字段类型：表示结构体字段的具体类型

    示例，定义一个Person结构体，代码如下：

    type person struct {
    name string
    age int8
    gender string
}

    同类型的字段也可以写在一行

    type person struct {
    name, gender string
    age int8
}

    这样我们就拥有了一个person的自定义类型，它有name、gender、age三个字段，分别表示姓名、性别、年龄，这样我们使用这个person结构体就能够很方便的在程序中表示和存储个人信息了

    Note

    语言内置的基础数据类型是用来描述一个值的，而结构体是用来描述一组值的。比如一个人有姓名、性别、年龄等信息，本质上是一种聚合型的数据类型

    实例化¶

    普通实例化¶

    只有当结构体实例化时，才会真正的分配内存，也就是必须实例化后才能使用结构体的字段

    结构体本身也是一种类型，我们可以像声明内置类型一样使用var关键字声明结构体类型

    var 结构体实例 结构体类型

    示例：

    package main

import "fmt"


// 定义person结构体
type person struct {
    name, gender string
    age          int8
}

func main() {
    // 实例化结构体
    var p1 person

    // 为结构体实例p1赋值
    p1.name = "小王子"
    p1.gender = "男"
    p1.age = 18

    // 打印p1的值
    fmt.Printf("p1: %v\n", p1)
    // %#v: 打印结构体时，会添加字段名
    fmt.Printf("p1: %#v\n", p1)
}
    执行结果
    p1: {小王子 男 18}
p1: main.person{name:"小王子", gender:"男", age:18}

    指针类型结构体实例化¶

    可以通过new关键字对结构体进行实例化，得到的是结构体的地址，语法格式如下：

    package main

import "fmt"


// 定义person结构体
type person struct {
    name, gender string
    age          int8
}

func main() {
    // 实例化结构体
    // 指针类型结构体实例化
    var p2 = new(person)
    fmt.Printf("p2: %T\n", p2)
    fmt.Printf("p2: %#v\n", p2)

    // 赋值
    p2.name = "孙悟空"
    p2.gender = "男"
    p2.age = 50

    // 打印p2的值
    fmt.Printf("p2: %#v\n", p2)

}
    执行结果
    p2: *main.person
p2: &main.person{name:"", gender:"", age:0}
p2: &main.person{name:"孙悟空", gender:"男", age:50}

    从打印的结果中，我们可以看出p2是一个结构体指针

    需要注意的是，在Golang中支持对结构体指针直接使用.来访问、赋值的

    取结构体地址实例化¶

    使用&对结构体进行取地址操作，相当于对该结构体类型进行了一次new实例化操作

    package main

import "fmt"


// 定义person结构体
type person struct {
    name, gender string
    age          int8
}

func main() {
    // 实例化结构体
    // 取结构体的地址实例化
    p3 := &person{}
    fmt.Printf("p3: %T\n", p3)
    fmt.Printf("p3: %#v\n", p3)

    // 赋值
    p3.name = "猪八戒"
    p3.age = 100
    p3.gender = "男"

    // 打印p3的值
    fmt.Printf("p3: %#v\n", p3)

}
    执行结果
    p3: *main.person
p3: &main.person{name:"", gender:"", age:0}
p3: &main.person{name:"猪八戒", gender:"男", age:100}

    p3.name =在底层其实是(*p3).name =，这是Golang帮我们实现的语法糖

    初始化¶

    结构体在实例化时可以直接对成员变量进行初始化，初始化有两种形式，分别是以字段"键值对"形式和多个值的列表形式

    Note

    键值对形式的初始化适合选择性填充字段较多的结构体

    多个值的列表形式适合填充字段较少的结构体

    使用键值对初始化¶

    结构体可以使用"键值对"初始化字段，每个"键"（key）对应结构体中的字段，值（value）对应该字段的初始值。

    package main

import "fmt"


// 定义person结构体
type person struct {
    name, gender string
    age          int8
}

func main() {
    // 使用键值对初始化结构体
    p4 := person{
        name:   "沙和尚",
        age:    20,
        gender: "男",
    }

    // 打印p4的值
    fmt.Printf("p4: %#v\n", p4)

    // 也可以对结构体指针进行键值对初始化
    p5 := &person{
        name:   "唐僧",
        age:    30,
        gender: "男",
    }

    // 打印p5的值
    fmt.Printf("p5: %#v\n", p5)


    // 某些字段没有初始值的时候, 该字段可以省略, 没有指定初始值的字段会被初始化为该类型的零值
    p8 := person{
        name: "牛魔王",
    }
    fmt.Printf("p8: %#v\n", p8)
}
    执行结果
    p4: main.person{name:"沙和尚", gender:"男", age:20}
p5: &main.person{name:"唐僧", gender:"男", age:30}
p8: main.person{name:"牛魔王", gender:"", age:0}

    使用值的列表初始化¶

    初始化结构体的时候可以简写，也就是不写键，直接写值

    package main

import "fmt"


// 定义person结构体
type person struct {
    name, gender string
    age          int8
}

func main() {
    // 结构体也可以简写
    p6 := &person{
        "白龙马",
        "男",
        100,
    }
    p7 := person{
        "红孩儿",
        "男",
        100,
    }
    fmt.Printf("p6: %#v\n", p6)
    fmt.Printf("p7: %#v\n", p7)
}
    执行结果
    p6: &main.person{name:"白龙马", gender:"男", age:100}
p7: main.person{name:"红孩儿", gender:"男", age:100}

    Warning

    使用值的列表方式初始化时，需要注意：

    • 必须初始化结构体的所有字段

    • 初始值的填充顺序必须与字段在结构体中的声明顺序一致

    • 该方式不能和键值初始化方式混用

    匿名结构体¶

    在定义一些临时数据结构等场景下还可以使用匿名结构体

    package main

import "fmt"

func main() {
    // 匿名结构体
    var user struct {
        name string
        age  int8
    }

    // 赋值
    user.age = 18
    user.name = "小王子"
    fmt.Printf("user: %#v\n", user)
}
    执行结果
    user: struct { name string; age int8 }{name:"小王子", age:18}

    空结构体¶

    空结构体语法格式：

    type Empty struct{}

    空结构体有什么用呢？

    var s Empty
fmt.Println(unsafe.Sizeof(s)) // 0

    可以看到，输出的空结构体类型变量的大小为0，也就是说，空结构体类型变量的内存占用为0。基于空结构体类型内存零开销这样的特性，我们在日常Go开发中会经常使用空结构体类型元素，作为一种"事件"信息进行Gotoutine之间的通信：

    var c = make(chan Empty)    // 声明一个元素类型为Empty的channel
c <- Empty{}        // 向channel中写入一个"事件"

    构造函数¶

    Golang中的结构体没有构造函数，我们可以自己实现，例如：下面的代码就实现了一个person的构造函数，因为struct是值类型，如果结构体比较复杂的话，值拷贝性能开销会比较大，所以该构造函数返回的是结构体指针类型：

    package main

import "fmt"

// 定义person结构体
type person struct {
    name, gender string
    age          int8
}

// 定义构造函数
func newPerson(name, gender string, age int8) *person {
    return &person{
        name:   name,
        gender: gender,
        age:    age,
    }
}

func main() {
    // 构造函数
    p9 := newPerson("猪八戒", "男", 100)
    fmt.Printf("p9: %#v\n", p9)
}
    执行结果
    p9: &main.person{name:"猪八戒", gender:"男", age:100}

    方法与接收者¶

    Golang中的方法（method）是一种作用于特定类型变量的函数。这种特定类型变量叫做接收者（Receiver），接收者的概念就类似于其它语言中的this或者self

    方法的语法格式如下：

    func (接收者变量 接收者类型) 方法名(参数列表) (返回参数){
    函数体
}

    Note

    其中：

    • 接收者变量：接收者中的参数变量在命名时，官方建议使用接收者类型名后的第一个小写字母，而不是self或者this
    • 接收者类型：接收者类型和参数类似，可以是指针类型和非指针类型
    • 方法名、参数列表、返回参数：具体格式与函数定义相同

    示例：

    package main

import "fmt"

// 定义person结构体
type person struct {
    name, gender string
    age          int8
}

// 定义构造函数
func newPerson(name, gender string, age int8) *person {
    return &person{
        name:   name,
        gender: gender,
        age:    age,
    }
}

// 定义Person 做梦的方法
func (p person) Dream() {
    fmt.Printf("%s的梦想是学好Go语言!\n", p.name)
}


func main() {
    // 构造函数
    p9 := newPerson("猪八戒", "男", 100)
    fmt.Printf("p9: %#v\n", p9)
    // 方法与接收者
    p9.Dream()
}
    执行结果
    p9: &main.person{name:"猪八戒", gender:"男", age:100}
猪八戒的梦想是学好Go语言!

    Note

    方法与函数的区别是，函数不属于任何类型，方法属于特定类型

    指针类型的接收者¶

    指针类型的接收者由一个结构体的指针组成，由于指针的特性，调用方法时修改接收者指针的任意成员变量，在方法接收后，修改都是有效的。

    这种方式十分接近其他语言中的self和this，例如：我们为person添加一个SetAge()方法，修改实例变量的年龄

    package main

import "fmt"

// 定义person结构体
type person struct {
    name, gender string
    age          int8
}

// 定义构造函数
func newPerson(name, gender string, age int8) *person {
    return &person{
        name:   name,
        gender: gender,
        age:    age,
    }
}

// 定义Person 做梦的方法
func (p person) Dream() {
    fmt.Printf("%s的梦想是学好Go语言!\n", p.name)
}

func (p *person) SetAge(newAge int8) {
    // 修改年龄
    p.age = newAge
}


func main() {
    // 构造函数
    p9 := newPerson("猪八戒", "男", 100)
    fmt.Printf("p9: %#v\n", p9)
    // 方法与接收者
    p9.Dream()

    // 调用方法修改年龄
    p9.SetAge(20)
    fmt.Printf("p9: %#v\n", p9)
}
    执行结果
    p9: &main.person{name:"猪八戒", gender:"男", age:100}
猪八戒的梦想是学好Go语言!
p9: &main.person{name:"猪八戒", gender:"男", age:20}

    值类型的接收者¶

    当方法作用于值类型接收者时，Golang会在代码运行时将接收者的值复制一份，在值类型接收者的方法中可以获取接收者的成员值，但修改操作只是针对副本，无法修改接收者变量本身

    package main

import "fmt"

// 定义person结构体
type person struct {
    name, gender string
    age          int8
}

// 定义构造函数
func newPerson(name, gender string, age int8) *person {
    return &person{
        name:   name,
        gender: gender,
        age:    age,
    }
}

// 定义Person 做梦的方法
func (p person) Dream() {
    fmt.Printf("%s的梦想是学好Go语言!\n", p.name)
}

func (p *person) SetAge(newAge int8) {
    // 修改年龄
    p.age = newAge
}

// 值类型接收者
func (p person) SetAge2(newAge int8) {
    // 修改年龄
    p.age = newAge
}


func main() {
    // 构造函数
    p9 := newPerson("猪八戒", "男", 100)
    fmt.Printf("p9: %#v\n", p9)
    // 方法与接收者
    p9.Dream()

    // 调用方法修改年龄
    p9.SetAge(20)
    fmt.Printf("p9: %#v\n", p9)

    p9.SetAge2(30)
    fmt.Printf("p9: %#v\n", p9)
}
    执行结果
    p9: &main.person{name:"猪八戒", gender:"男", age:100}
猪八戒的梦想是学好Go语言!
p9: &main.person{name:"猪八戒", gender:"男", age:20}
p9: &main.person{name:"猪八戒", gender:"男", age:20}

    Note

    什么时候使用指针接收者？

    • 需要修改接收者中的值
    • 接收者是拷贝代价比较大的大对象
    • 保证一致性，如果有某个方法使用了指针接收者，那么其他方法也应该使用指针接收者

    任意类型添加方法¶

    在Golang中，接收者的类型可以是任何类型，不仅仅是结构体，任何类型都可以拥有方法。

    例如：我们在内置的int类型使用type关键字可以定义新的自定义类型，然后为自定义类型添加方法

    package main

import "fmt"

type MyInt int

// 为自定义类型添加方法
func (m MyInt) SayHello() {
    fmt.Println("Hello, 我是一个int!")
}


func main() {
    // 实例化自定义类型
    var i MyInt
    // 调用方法
    i.SayHello()
    i = 100
    fmt.Printf("i: %v, type: %T\n", i, i)
}
    执行结果
    Hello, 我是一个int!
i: 100, type: main.MyInt

    结构体匿名字段¶

    结构体允许成员字段在声明时没有字段名而只有类型，这种没有名字的字段就叫做匿名字段

    package main

import "fmt"

// 匿名字段
type Person struct {
    name string
    string
    int
}

func main() {
    p1 := Person{"小王子", "北京", 18}
    fmt.Printf("p1: %#v\n", p1)
    fmt.Println(p1.name, p1.string, p1.int)
}
    执行结果
    p1: main.Person{name:"小王子", string:"北京", int:18}
小王子 北京 18

    Note

    匿名字段默认采用类型名作为字段名，结构体要求字段名称必须唯一，因此一个结构体中同种类型的匿名字段只能有一个

    结构体嵌套¶

    普通结构体嵌套¶

    一个结构体中可以嵌套包含另一个结构体或者结构体指针

    package main

import "fmt"

// 结构体的嵌套
type Address struct {
    Province, City string
}

type User struct {
    Name, Gender string
    Age          int
    Address      Address
}

func main() {

    user1 := User{
        Name:   "小王子",
        Gender: "男",
        Age:    18,
        Address: Address{
            Province: "北京",
            City:     "北京",
        },
    }
    fmt.Printf("user1: %#v\n", user1)

}
    执行结果
    user1: main.User{Name:"小王子", Gender:"男", Age:18, Address:main.Address{Province:"北京", City:"北京"}}

    匿名结构体嵌套¶

    package main

import "fmt"

// 结构体的嵌套
type Address struct {
    Province, City string
}

type User struct {
    Name, Gender string
    Age          int
    // Address      Address
    Address // 匿名嵌套结构体
}

func main() {

    user1 := User{
        Name:   "小王子",
        Gender: "男",
        Age:    18,
    }

    // 匿名嵌套结构体赋值
    user1.Address.Province = "北京" // 通过匿名结构体.字段名 赋值
    user1.City = "北京"             // 直接通过匿名结构体字段名赋值

    fmt.Printf("user1: %#v\n", user1)

}
    执行结果
    user1: main.User{Name:"小王子", Gender:"男", Age:18, Address:main.Address{Province:"北京", City:"北京"}}

    当访问结构体成员时，会现在结构体中寻找，找不到再去匿名结构体中查找

    嵌套结构体的字段名冲突¶

    嵌套结构体内部可能存在相同的字段名，这个时候为了避免歧义需要指定具体的内嵌结构体字段

    package main

import "fmt"


// 结构体的嵌套
type Address struct {
    Province, City, CreateTime string
}

type Email struct {
    Account, CreateTime string
}

type User struct {
    Name, Gender string
    Age          int
    Address // 匿名嵌套结构体
    Email   // 匿名嵌套结构体
}

func main() {
    user1 := User{
        Name:   "小王子",
        Gender: "男",
        Age:    18,
    }

    // 匿名嵌套结构体赋值
    user1.Address.Province = "北京"           // 通过匿名结构体.字段名 赋值
    user1.City = "北京"                       // 直接通过匿名结构体字段名赋值
    user1.Address.CreateTime = "2020-01-01" // 指定Address结构体字段赋值, 不会影响Email结构体字段

    user1.Email.Account = "xwz"
    user1.Email.CreateTime = "2020-01-02"

    fmt.Printf("user1: %#v\n", user1)

}
    执行结果
    user1: main.User{Name:"小王子", Gender:"男", Age:18, Address:main.Address{Province:"北京", City:"北京", CreateTime:"2020-01-01"}, Email:main.Email{Account:"xwz", CreateTime:"2020-01-02"}}

    结构体继承¶

    Golang中使用结构体也可以实现其他语言中面向对象的继承

    package main

import "fmt"


// 定义动物结构体
type Animal struct {
    Name string
}

// 定义动物移动的方法
func (a *Animal) Move() {
    fmt.Printf("%s会动!\n", a.Name)
}

// 定义狗结构体
type Dog struct {
    Feet    int8
    *Animal // 匿名嵌套结构体
    // 不嵌套指针也可以
    // Animal
}

// 定义狗叫的方法
func (d *Dog) Wang() {
    fmt.Printf("%s会汪汪汪!\n", d.Name)
}

func main() {
    // 结构体的继承
    d1 := Dog{
        Feet: 4,
        Animal: &Animal{ // 注意嵌套的是结构体指针类型
            Name: "旺财",
        },
        // Animal: Animal{
        //    Name: "旺财"
        // }
    }
    // 调用狗的方法
    d1.Wang()
    // 调用动物的方法
    d1.Move()

}
    执行结果
    旺财会汪汪汪!
旺财会动!

    结构体与JSON序列化¶

    JSON是一种轻量级的数据交换格式，易于人们阅读和编写，同时也易于机器解析和生成，JSON键值对是用来保存JS对象的一种方式，键/值对组合中的键名写在前面并用双引号""包裹，使用冒号:分隔，然后紧接着值；多个键值之间使用英文,分隔

    package main

import (
    "encoding/json"
    "fmt"
)


// JSON与结构体
type Student struct {
    ID     int
    Gender string
    Name   string
}

type Class struct {
    Title    string
    Students []*Student
}

func main() {
    // JSON与结构体
    // 初始化结构体, &Class: 返回结构体指针
    c := &Class{
        Title: "101",
        // Students: make([]*Student, 0, 200), // 初始化切片
        Students: make([]*Student, 0, 200),
    }

    for i := 0; i < 10; i++ {
        // 初始化结构体, &Student: 返回结构体指针
        stu := &Student{
            ID:     i,
            Gender: "男",
            Name:   fmt.Sprintf("stu%02d", i),
        }
        // 追加到Students切片中
        c.Students = append(c.Students, stu)
    }

    fmt.Printf("c: %#v\n", c)

    // JSON序列化
    // json.Marshal()函数: 序列化结构体, 返回[]byte切片和error
    data, err := json.Marshal(c)
    if err != nil {
        fmt.Println("json marshal failed!")
        return
    }
    fmt.Printf("json: %s\n", data)

    // JSON反序列化: 将JSON字符串反序列化为结构体
    str := `{"Title":"101","Students":[{"ID":0,"Gender":"男","Name":"stu00"},{"ID":1,"Gender":"男","Name":"stu01"},{"ID":2,"Gender":"男","Name":"stu02"},{"ID":3,"Gender":"男","Name":"stu03"},{"ID":4,"Gender":"男","Name":"stu04"},{"ID":5,"Gender":"男","Name":"stu05"},{"ID":6,"Gender":"男","Name":"stu06"},{"ID":7,"Gender":"男","Name":"stu07"},{"ID":8,"Gender":"男","Name":"stu08"},{"ID":9,"Gender":"男","Name":"stu09"}]}`

    // 初始化结构体, &Class: 返回结构体指针
    c1 := &Class{}

    // json.Unmarshal()函数: 反序列化JSON字符串, 返回error
    err = json.Unmarshal([]byte(str), c1)

    if err != nil {
        fmt.Println("json unmarshal failed!")
        return
    }
    fmt.Printf("c1: %#v\n", c1)

}
    执行结果

    结构体标签Tag¶

    Tag是结构体的元信息，可以在运行的时候通过反射的机制读取

    Tag在结构体字段的后方定义，由一对反引号包裹起来，具体的格式如下：

    `k1:"v1" k2:"v2"`

    结构体标签由一个或多个键值对组成。键与值使用冒号:分隔，值用双引号""括起来，键值对之间使用一个空格分隔。

    注意事项：

    • 为结构体编写Tag时，必须严格遵守键值对规则，例如：不要在key和value中间加空格
    • 结构体标签的解析代码的容器能力很差，一旦格式写错，编译和运行时都不会提示任何错误，通过反射也无法获取到正确值
    package main

import (
    "encoding/json"
    "fmt"
)


// 结构体标签
type Student struct {
    ID     int    `json:"id"` //通过指定tag实现json序列化该字段时的key
    Gender string // 没有指定tag，json序列化时key默认为字段名
    name   string // 首字母小写，私有字段，json不会序列化
}

func main() {
    // 结构体标签
    s1 := Student{
        ID:     1,
        Gender: "男",
        name:   "小王子",
    }
    // json序列化
    data, err := json.Marshal(s1)

    if err != nil {
        fmt.Println("json marshal failed!")
        return
    }
    fmt.Printf("json: %s\n", data)

}
    执行结果
    json: {"id":1,"Gender":"男"}
