    函数¶

    函数是Golang中的 一级公民，我们把所有的功能单元都定义在函数中，可以重复使用，函数包含函数的名称、参数列表和返回值类型，这些构成了函数的签名

    函数特点¶

    • Golang有三种函数：普通函数、匿名函数（没有名称的函数）、方法（定义在结构体上的函数）
    • Golang中不允许函数重载，也就是不允许函数同名
    • 函数不能嵌套函数，但可以嵌套匿名函数
    • 函数是一个值，可以将函数赋值给变量，使得这个变量也成为函数
    • 函数可以作为参数传递给另一个函数
    • 函数的返回值可以是一个函数
    • 函数调用的时候，如果有参数传递给函数，则先拷贝参数的副本，再将副本传递给函数
    • 函数参数可以没有名称
    • 函数不支持默认参数

    函数定义¶

    定义语法：

    func function_name([param_list]) [return_types] {
    函数体
}

    语法解析

    • func：函数由func关键字声明
    • function_name：函数名称
    • [param_list]：参数列表，参数就像一个占位符，当函数被调用时，你可以将值传递给参数，这个值被称为实际参数。参数列表指定的是参数类型、顺序、以及参数个数。参数是可选的。
    • [return_types]：返回类型，函数返回一列值，return_types是该列值得数据类型。返回类型是可选的。
    • 函数体：函数定义的代码集合

    函数实例¶

    求和函数¶

    package main

// 定义一个求和函数
func sum(a, b int) int {
    return a + b
}

func main() {
    // 函数也是一种数据类型，可以赋值给一个变量
    f := sum
    // 调用函数
    println(f(10, 20))
}
    执行结果
    30

    比较大小函数¶

    package main

// 定义一个求和函数
func sum(a, b int) int {
    return a + b
}

// 定义一个比较大小的函数
func compare(a, b int) int {
    if a > b {
        return a
    }
    return b
}

func main() {
    // 函数也是一种数据类型，可以赋值给一个变量
    f := sum
    // 调用函数
    println(f(10, 20))

    // 函数也可以作为参数传递给另一个函数
    println(compare(sum(10, 20), 40))
}
    执行结果
    30
40

    函数返回值¶

    函数可以有0个或者多个返回值，返回值需要指定数据类型，返回值通过return关键字来指定

    return可以有参数，也可以没有参数，这些返回值可以有名称，也可以没有名称

    • return关键字中指定了参数时，返回值可以不用名称。如果return省略参数，则返回值部分必须带名称
    • 当返回值有名称时，必须使用括号包围，逗号分隔，即使只有一个返回值
    • 但即使返回值命名了，return中也可以强制指定其他返回值的名称，也就是说return的优先级更高
    • 命名的返回值是预先声明好的，在函数内部可以直接使用，无需再次声明。命名返回值的名称不能和函数参数名称相同，否则会报错提示变量重复定义
    • return中可以有表达式，但不能出现赋值表达式，这和其他语言可能有所不同，例如return a + b是正确的，但是return c=a+b是错误的

    没有返回值的函数¶

    func f1() {
    fmt.Println("我没有返回值")
}
    执行结果
    我没有返回值

    有一个返回值¶

    func sum(a, b int) (ret int) {
    ret = a + b
    return ret
}
fmt.Printf("sum: %v\n", sum1(10, 20))
    执行结果
    sum: 30

    有多个返回值，且在return中指定返回内容¶

    func f2() (name string, age int) {
    name = "cxy"
    age = 20
    return name, age
}

name, age := f2()
fmt.Printf("name: %v, age: %v", name, age)
    执行结果
    name: cxy, age: 18

    有多个返回值，返回值名称没有被使用¶

    func f3() (name string, age int) {
    name = "cxy"
    age = 20 
    return // 相当于return name, age
}


name, age = f3()
fmt.Printf("name: %v, age: %v\n", name, age)
    执行结果
    name: cxy, age: 20

    return覆盖命名返回值，返回值名称没有被使用¶

    func f4() (name string, age int) {
    n := "cxy"
    a := 15
    return n, a
}

name, age = f4()
fmt.Printf("name: %v, age: %v\n", name, age)
    执行结果
    name: cxy, age: 15

    Warning

    Go中经常会使用其中一个返回值作为函数是否执行成功、是否有错误信息的判断条件。例如return value, exists、return value, ok、return value, err等

    当函数的返回值过多时，例如有4个及以上的返回值，就应该将这些返回值收集到容器中，然后以返回容器的方式返回信息。例如，同类型的返回值可以放进slice中，不同类型的返回值可以放进map中

    当函数有多个返回值时，如果其中某个或者某几个返回值不想使用，可以通过下划线_来丢弃这些返回值

    函数参数¶

    Golang函数可以有0个或者多个参数，参数需要指定数据类型。

    声明函数时的参数列表叫做形参，调用时传递的参数叫做实参

    Golang是通过 传值的方式传参，意味着传递给函数的是拷贝后的副本，并非原变量，所以函数内部访问、修改的也是这个副本

    Golang可以使用 变长参数，有时候不能确定参数的个数，可以使用变长参数，可以在函数定义语句的参数部分使用ARGS...TYPE的方式，这时会将...代表的参数全部保存到一个名为ARGS的slice中，注意这些参数的数据类型都是TYPE。

    值传递¶

    值传递：在调用函数时，将实际参数复制一份到函数中，这样在函数中如果对参数进行修改，不会影响到实际参数

    package main

import "fmt"

func f5(a int) {
    a = 200
    fmt.Printf("a: %v\n", a)
}

func main() {
    a := 100
    f5(a)
    fmt.Printf("a: %v\n", a)
}
    执行结果
    a: 200
a: 100

    从运行结果可以看到，调用函数f5后，a的值并没有被改变。

    Note

    map、slice、interface、channel这些数据类型本身就是 指针 类型，所以就算是拷贝传值也是拷贝的指针，拷贝后的参数仍然指向底层数据结构，所以修改它们 可能 会影响外部数据结构的值

    package main

import "fmt"

func f6(a []int) {
    a[0] = 200
    fmt.Printf("a: %v\n", a)
}

func main() {
    a := []int{1, 2, 3, 4, 5}
    fmt.Printf("未调用函数前a的值: %v\n", a)

    f6(a)
    fmt.Printf("调用函数后a的值: %v\n", a)
}
    执行结果
    未调用函数前a的值: [1 2 3 4 5]
a: [200 2 3 4 5]
调用函数后a的值: [200 2 3 4 5]

    Warning

    那么，什么时候对slice等这些类型进行值传递时是不会影响原变量值呢？

    当在函数中对slice进行操作，超出了原本的cap限制，此时会重新分配底层数组，也就不会影响到原变量值了

    package main

import "fmt"

func f6(a []int) {
    a = append(a, 6)
    a[0] = 100
    fmt.Printf("a: %v\n", a)
}

func main() {
    a := []int{1, 2, 3, 4, 5}
    fmt.Printf("未调用函数前a的值: %v\n", a)

    f6(a)
    fmt.Printf("调用函数后a的值: %v\n", a)
}
    执行结果
    未调用函数前a的值: [1 2 3 4 5]
a: [100 2 3 4 5 6]
调用函数后a的值: [1 2 3 4 5]

    引用传递¶

    引用传递：指在调用函数时将实际参数的地址传递到函数中，那么在函数中对参数所进行的修改，将会影响实际参数

    package main

import "fmt"

// 引用传递
func f6(x, y *int) {
    var temp int
    temp = *x // 保存x的值
    *x = *y   // 将y的值赋值给x
    *y = temp // 将x的值赋值给y
}

func main() {
    // 引用传递
    var a, b int = 10, 20
    // 调用f6函数
    f6(&a, &b)
    fmt.Printf("a: %v, b: %v\n", a, b)
}
    执行结果
    a: 20, b: 10

    Note

    在默认情况下，Golang使用的是值传递，即在调用过程中不会影响到实际参数。

    • 无论是值传递，还是引用传递，传递给函数的都是变量的副本，不过，值传递是值的拷贝。引用传递是地址的拷贝，一般来说，地址拷贝更为高效。而值拷贝取决于拷贝的对象大小，对象越大，则性能越低
    • map、slice、chan、指针、interface默认以引用方式传递

    不定参数传值¶

    不定参数传值就是函数的参数不是固定的，后面的类型是固定的

    Golang可变参数本质上就是slice，只能有一个，且位置必须是在最后

    在参数赋值时可以不用一个一个的赋值，可以直接传递一个数组或者切片，特别注意的是在参数后加上...即可

    func myfunc(args ...int) {}  //0个或多个参数

func myfunc(a int, args ...int) int {} // 1个或者多个参数

func myfunc(a, b int, args ...int) int {} // 2个或者多个参数

    Warning

    注意：其中args是一个slice，我们可以通过arg[index]依次访问所有参数，通过len(arg)来判断传递参数的个数

    package main

import "fmt"

// 不定参
func f7(s string, n ...int) string {
    // 定义x变量
    var x int
    // 遍历n
    for _, v := range n {
        x += v
    }
    // 返回结果
    return fmt.Sprintf(s, x)
}

func main() {
    s := f7("sum: %v\n", 1, 2, 3, 4, 5)
    fmt.Println(s)
}
    执行结果
    sum: 15

    如果使用slice对象做变参时，必须要展开

    package main

import (
    "fmt"
)

func test(s string, n ...int) string {
    var x int
    for _, i := range n {
        x += i
    }

    return fmt.Sprintf(s, x)
}

func main() {
    s := []int{1, 2, 3}
    res := test("sum: %d", s...)    // slice... 展开slice
    println(res)
}

    函数类型和函数变量¶

    可以使用type关键字来定义一个函数类型，语法格式如下：

    type fun func(int, int) int

    上面语句定义了一个fun函数类型，它是一种函数类型，这种函数接收两个int类型的参数并且返回一个int类型的返回值

    下面我们定义两个这样结构的函数，一个求和，一个比较大小

    func sum(a, b int) int {
    return a + b
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

    下面定义一个fun函数类型，把sum和max赋值给它

    package main

import (
    "fmt"
)

func sum(a, b int) int {
    return a + b
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

// 定义一个函数类型
type fun func(int, int) int

func main() {
    // 实例化一个函数类型
    var f fun
    f = sum
    fmt.Printf("f: %v\n", f(10, 20))

    f = max
    fmt.Printf("f: %v\n", f(10, 20))
}
    执行结果
    f: 30
f: 20

    高阶函数¶

    Golang中函数可以作为函数的参数，传递给另外一个函数，也可以作为另一个函数的返回值

    函数作为参数¶

    package main

import (
    "fmt"
)

// 将函数作为参数传递给另一个函数
func sayHello(name string) {
    fmt.Printf("Hello, %v\n", name)
}

func f1(name string, f func(string)) {
    f(name)
}

func main() {
    // 将函数作为参数传递给另一个函数
    f1("cxy", sayHello)
}
    执行结果
    Hello, cxy

    函数作为返回值¶

    package main

import (
    "fmt"
)

// 将函数作为返回值
func add(x, y int) int {
    return x + y
}

func sub(x, y int) int {
    return x - y
}

func cal(s string) func(int, int) int {
    switch s {
    case "+":
        return add
    case "-":
        return sub
    default:
        return nil
    }
}

func main() {
    // 将函数作为返回值
    add := cal("+")
    r := add(10, 20)
    fmt.Printf("r: %v\n", r)

    sub := cal("-")
    r = sub(100, 20)
    fmt.Printf("r: %v\n", r)
}
    执行结果
    r: 30
r: 80

    匿名函数¶

    Golang中函数不能嵌套，但是在函数内部可以定义匿名函数，实现简单功能调用。

    所谓匿名函数，就是没有名称的函数。

    匿名函数语法格式如下：

    func (参数列表)(返回值){
    结构体
}

    Note

    匿名函数可以既没有参数，也没有返回值

    匿名函数实例¶

    package main

import (
    "fmt"
    "math"
)

func main() {
    max := func(a, b int) int {
        if a > b {
            return a
        }
        return b
    }

    i := max(10, 20)

    fmt.Printf("i: %v\n", i)
}
    执行结果
    i: 20

    上面显示定义了一个名为max的变量，初始化该变量和之前的变量初始化有所不同，使用了func，func是定义函数的，可是这个函数和之前的函数相比没有函数名，被称为匿名函数。这里将一个函数当作一个变量操作。

    自己调用自己¶

    package main

import (
    "fmt"
    "math"
)

func main() {
    // 自己调用自己
    func(a, b int) {
        max := 0
        if a > b {
            max = a
        } else {
            max = b
        }
        fmt.Printf("max: %v\n", max)
    }(10, 20)
}
    执行结果
    max: 20

    闭包¶

    闭包可以理解成 定义在一个函数内部的函数，在本质上，闭包就是将函数内部和函数外部连接起来的桥梁，或者说是函数和其引用环境的组合体

    闭包指的是一个函数与其相关的引用环境组合而成的实体。简单来说，闭包=函数+引用环境。

    package main

import "fmt"


// 闭包函数
func add() func(int) int {
    var x int
    return func(y int) int {
        x += y
        return x
    }
}

func main() {
    // 闭包
    var f = add()
    fmt.Printf("f(10): %v\n", f(10))
    fmt.Printf("f(20): %v\n", f(20))
    fmt.Printf("f(30): %v\n", f(30))

    fmt.Println("-----------------")

    f1 := add()
    fmt.Printf("f1(40): %v\n", f1(40))
    fmt.Printf("f1(50): %v\n", f1(50))
}
    执行结果
    f(10): 10
f(20): 30
f(30): 60
-----------------
f1(40): 40
f1(50): 90

    变量f是一个函数，并且它引用了其外部作用域中的x变量，此时f就是一个闭包，在f的生命周期内，变量x也一直有效。

    闭包进阶：

    package main

import "fmt"


// 闭包函数
func makeSuffixFunc(suffix string) func(string) string {
    return func(name string) string {
        // 判断name是否以传进来的字符串结尾
        if !strings.HasSuffix(name, suffix) {
            return name + suffix
        }
        return name
    }
}


func main() {
    // 闭包
    jpgFunc := makeSuffixFunc(".jpg")
    txtFunc := makeSuffixFunc(".txt")
    fmt.Printf("jpgFunc: %v\n", jpgFunc("test"))
    fmt.Printf("txtFunc: %v\n", txtFunc("test"))
}
    执行结果
    jpgFunc: test.jpg
txtFunc: test.txt

    闭包进阶：

    package main

import "fmt"


// 闭包函数
// calc(base int) (func(int) int, func(int) int): 传入一个int类型整数, 返回两个函数, 两个函数都需要传入一个int类型参数, 然后返回一个int类型返回值
func calc(base int) (func(int) int, func(int) int) {
    add := func(i int) int {
        base += i
        return base
    }
    sub := func(i int) int {
        base -= i
        return base
    }
    return add, sub
}


func main() {
    // 闭包
    f1, f2 := calc(10)
    fmt.Printf("f1(1): %v, f2(2): %v\n", f1(1), f2(2))
    fmt.Printf("f1(3): %v, f2(4): %v\n", f1(3), f2(4))
    fmt.Printf("f1(5): %v, f2(6): %v\n", f1(5), f2(6))
}
    执行结果
    f1(1): 11, f2(2): 9
f1(3): 12, f2(4): 8
f1(5): 13, f2(6): 7

    递归¶

    函数内部调用函数自身的函数称为递归函数

    使用递归函数最重要的三点：

    • 递归就是自己调用自己
    • 必须先定义函数的退出条件，没有退出条件，递归就成为死循环
    • Golang递归函数很可能会产生一大堆的goroutine，也很可能会出现栈空间内存溢出问题

    构成递归需要具备的条件：

    • 子问题必须与原始问题为同样的事，且更为简单
    • 不能无限制的调用本身，必须设置退出条件

    阶乘¶

    package main

import "fmt"


// 递归函数
func a(n int) int {
    // 退出条件
    if n == 1 {
        return 1
    } else {
        // 调用自身
        fmt.Printf("%v*%v\n", n, n-1)
        return n * a(n-1)
    }
}


func main() {
    i := a(5)
    fmt.Printf("i: %v\n", i)
}
    执行结果
    5*4
4*3
3*2
2*1
i: 120

    斐波那契数列¶

    package main

import "fmt"


// 递归函数
// 斐波那契数列
func fibonaci(i int) int {
    if i == 0 {
        return 0
    } else if i == 1 {
        return 1
    }
    return fibonaci(i-1) + fibonaci(i-2)
}



func main() {
    for i := 0; i < 10; i++ {
        fmt.Printf("fibonaci(%v): %v\n", i, fibonaci(i))
    }
}

    defer¶

    Golang中的defer语句会将其后面跟随的语句进行 延迟 处理，在defer归属的函数即将返回时，将延迟处理的语句按照defer定义的逆序进行执行，也就是说，先被defer的语句最后被执行，最后被defer的语句最先被执行

    defer特性¶

    • 关键字defer用于注册延迟调用
    • 这些调用直到return前才被执行，因此，可以用来做资源清理
    • 多个defer语句，按先进后出的方式执行
    • defer语句中的变量，在defer声明时就决定了

    defer用途¶

    • 关闭文件句柄
    • 锁资源释放
    • 数据库连接释放

    Warning

    defer为什么是先进后出？

    因为后面的语句会依赖前面的资源，如果前面的资源先释放了，后面的语句就没办法执行

    defer实例¶

    package main

import "fmt"

func main() {
    // defer
    a := []int{1, 2, 3, 4, 5}
    for i := range a {
        defer fmt.Printf("a[%v]: %v\n", i, a[i])
    }
}
    执行结果
    a[4]: 5
a[3]: 4
a[2]: 3
a[1]: 2
a[0]: 1
